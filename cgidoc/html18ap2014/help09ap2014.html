How to Use

Algorithm

The SubNucPred predictor first searches for Pfam domains in the query sequence. If a Pfam domain is found, the predictor directs the query sequence to Method-I where the location is predicted on the basis of presence of unique Pfam domain. If Method-I does not find any unique domain, the query is forwarded to Method-II, which uses amino acid composition based SVM modules for prediction.
For SVM based prediction, we adopted two-layered training approach. In first layer we kept location with more than 50 proteins viz (1) centromere (2) chromosome (3) nuclear speckle (4) nucleolus and (5) others containing all locations having less than 50 proteins (nuclear envelope, nuclear matrix, nucleoplasm, nuclear pore complex, PML body and telomere). In second layer, sequences belonging to locations 'others' were used only and SVM models were developed using the data of 'others' location only. In short, SubNucPred presents the query protein to layer-I and predicts it as a member of class which has maximum score. In case, the maximum score obtained was from 'Others' the query will be forwarded to layer-II. The figue 1 shows flow diagram of SubNucPred

Input Sequences

User can submit single as well as multiple sequences at a time in fasta format. If user submit less than or equal to 25 sequences, the server predict all the sequences but if user submit more than 25 sequences, the server predict only first 25 sequences. Prediction Result

When user submit the query, the predictin results comes on same window in a tabular form. If the query protein were predicted according to their unique Pfam domain, the result shows predicted location and domain based prediction in a table and if the query proteins predict according to SVM score the result shows predicted location and their corresponding SVM score.
